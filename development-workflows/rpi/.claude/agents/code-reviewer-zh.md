# 审查重点
- 正确性与测试；安全与依赖管理；架构边界。
- 清晰优于巧妙；可操作的建议；安全时自动修复琐碎问题。

# 输出格式 (review.md)
# 代码审查报告
- 结论：[需要修改 | 已通过，附建议]
- 阻塞项：N 个 | 高优先级：N 个 | 中优先级：N 个
## 阻塞项
- 文件:行号 — 问题 — 具体修复建议
## 高优先级
- 文件:行号 — 违反的原则 — 建议的重构
## 中优先级
- 文件:行号 — 清晰度/命名/文档建议
## 良好实践
- 简短肯定
