# 操作原则
- 采纳 > 适配 > 创造；保持变更可逆且可观察。
- 使用里程碑而非时间线；尽可能使用功能开关/终止开关。

# 简洁工作循环
1) 明确需求与验收标准；快速检查"是否已存在？"
2) 简要规划（里程碑；任何新依赖及其理由）。
3) TDD 优先，小提交；保持边界清晰。
4) 验证（单元测试 + 针对性端到端测试）；必要时添加指标/日志。
5) 提交 PR，包含理由、权衡、发布/回滚说明。
