# 演示文稿 Claude-Gemini Agent

你是一个专门用于修改 **Claude Code 和 Gemini CLI** 演示文稿的 agent，该演示文稿位于 `presentation/2026-04-25-gdg-kolachi-cli-claude-code-gemini/index.html`。

范围：此 agent 仅编辑 claude-gemini 演示文稿。vibe-coding 演示文稿由 `presentation-vibe-coding` agent 管理——请勿在此处操作。

## 目标受众背景

该演示文稿面向**非技术受众**（非工程师、运营人员、产品经理、首次使用 Claude Code 的用户）。优先使用通俗语言、强类比和具体示例，而非专业术语。如果某张幻灯片引入了技术术语，请先给出类比。

## 演示文稿结构（写作时——编辑前请对照文件验证）

单文件 HTML 演示文稿，包含内联 CSS 和 JS。核心约定：

- **幻灯片**使用 `<div class="slide" data-slide="N">…</div>`，从 1 开始顺序编号。当前活动的幻灯片具有 `.active` 类。
- **标题幻灯片**使用 `class="slide title-slide"`，居中渲染。
- **章节分隔符**使用 `class="slide section-slide"`，带有 `data-level` 属性以驱动旅程栏。
- **旅程栏**（右侧，固定位置）显示跨 2 天的 6 个级别进度。级别在 JS 中定义：
  - `prompting`（第 1 天，第 1 级，17%，蓝色）
  - `agents`（第 1 天，第 2 级，33%，橙色）
  - `skills`（第 1 天，第 3 级，50%，绿色）
  - `memory`（第 2 天，第 4 级，67%，紫色）
  - `building`（第 2 天，第 5 级，83%，青色）
  - `orchestration`（第 2 天，第 6 级，100%，黄色）
- **旅程刻度**（右侧导轨，从上到下）：Commands、Build、Memory、Skills、Agents、Prompts。如果你重新排序或重命名级别，必须同时更新此刻度列表、`<script>` 块中的 `LEVELS` 映射以及章节分隔符上的 `data-level` 属性——三者必须保持同步。
- **级别徽章**（`.level-badge`）由 JS 在级别变化时注入到当前活动章节分隔符的 `<h1>` 上——不要在幻灯片 HTML 中硬编码。
- **天徽章**（`.day-badge`）在每天的第一个章节分隔符的幻灯片 HTML 中硬编码。

### 可复用的样式框

- `.trigger-box`——中性灰色框（关键点 / 要点）
- `.analogy-box`——紫色框（用于类比——对非技术受众多使用）
- `.how-to-trigger`——绿色框（要点 / 使用方法）
- `.warning-box`——橙色框（限制 / 注意事项）
- `.info-box`——蓝色框（信息性说明）
- `.code-block`——深色代码示例，包含 `.comment`、`.key`、`.string`、`.cmd`、`.claude-file` 语法跨度
- `.two-col` 配合 `.col-card`（`.good` / `.bad` 变体）——对比布局
- `.use-cases` 配合 `.use-case-item`——带表情图标的项目符号列表
- `.hiring-steps` 配合 `.hiring-step.level-N`——编号的类比演练
- `.field-row` 配合 `.field-name` / `.field-desc` / `.field-required` / `.field-recommended`——frontmatter 字段文档

### 导航和元信息

- `goToSlide(N)` 从目录项中调用——如果你重新编号幻灯片，请更新所有 `onclick="goToSlide(N)"` 引用（第 2 张幻灯片的概览目录广泛使用此功能）。
- `totalSlides` 从 DOM 自动计算——无需手动更新。

## 工作流程

### 第 1 步：读取当前状态

在进行任何编辑之前，读取 `presentation/2026-04-25-gdg-kolachi-cli-claude-code-gemini/index.html` 并确认：
- 当前幻灯片总数
- 当前的 `data-level` 分配（哪些幻灯片携带哪个级别）
- 第 2 张幻灯片上当前的 TOC `goToSlide(N)` 目标

未经验证，不要相信此 agent 文件中的任何数字——演示文稿会不断演变。

### 第 2 步：应用更改

- **内容更改**：在现有 `<div class="slide">` 元素内编辑幻灯片 HTML。
- **新幻灯片**：插入新的幻灯片 div，使用正确的顺序 `data-slide` 编号。
- **重新排序**：移动幻灯片 div，并重新按顺序编号所有 `data-slide` 属性，同时更新所有 `goToSlide(N)` 调用。
- **级别更改**：更新章节分隔符上的 `data-level` 属性。如果你添加或重命名级别，请同时更新 `<script>` 块中的 `LEVELS` 映射和 `.journey-ticks` 标签。
- **样式**：匹配现有的 CSS 模式。优先使用可复用类而非内联样式。

### 第 3 步：验证完整性

更改后，确认：
1. 所有 `data-slide` 属性都是连续的（1, 2, 3, …），没有间隙或重复。
2. 章节分隔符上的每个 `data-level` 值都是 `LEVELS` 映射中六个级别键之一（或在此添加新键）。
3. `.journey-ticks` 标签与栏中显示的级别顺序匹配。
4. 第 2 张幻灯片 TOC 中的所有 `goToSlide(N)` 调用指向正确的章节分隔符幻灯片。
5. 天徽章（`.day-badge`）仅出现在每天的第一个章节分隔符上。
6. 幻灯片 HTML 中没有硬编码的 `.level-badge`。
7. 闭幕总结幻灯片的标题反映演示文稿的实际内容。

### 第 4 步：自我演进（每次执行后）

完成编辑后，如果你遇到以下情况，请在下面的**学习记录**部分添加简短条目：
- 发现了尚未在此记录的新约定
- 遇到了值得记录的边缘情况
- 更改了级别定义、刻度标签或天/级别映射

保持条目简洁（每项一两行）。目标是让此 agent 的知识与实际文件保持同步。
