## 核心要点

### 为未来构建，不为当下

- "在 Anthropic，我们不为今天的模型构建。我们为 6 个月后的模型构建"
- 这是 Boris 仍然给基于 LLM 构建的创始人的建议
- 思考"模型今天还不太擅长但即将变得擅长的前沿是什么"

### Claude Code 的持续重写

- "Claude Code 的每一部分都被一遍又一遍地重写。6 个月前存在的部分现在一个都不在了"
- 开发方法论：尝试、给用户、与用户交流、学习、最终可能找到一个好主意（有时不是）

### Plan Mode 可能很快消失

- 当被问到"6 个月后是否还需要显式地 prompting"，Boris 暗示"也许一个月后就不需要 plan mode 了"
- 模型的推理能力在快速提升，可能很快能自己弄清楚需求

### 兴奋与上瘾

- YC 主持人说："感谢你创造了一个让我连续 3 周失眠的东西。我对 Claude Code 非常上瘾，感觉像火箭助推器"
- Boris 回忆自己创建 Claude Code 时的感觉——不确定是否抓住了什么，但就是睡不着

### 转折点

- 2025 年 11 月底是 Boris 很多朋友说"某些东西变了"的时间点
- 模型的 agentic 能力达到了一个临界点

## 关键引用

- "We don't build for the model of today. We build for the model six months from now."（我们不为今天的模型构建。我们为 6 个月后的模型构建。）

- "All of Quad Code has just been written and rewritten and rewritten and rewritten over and over and over. There is no part of Quad Code that was around 6 months ago."（Claude Code 的每一部分都被一遍又一遍地重写。6 个月前存在的部分现在一个都不在了。）

- "Maybe in a month, no more need for plan mode."（也许一个月后，不再需要 plan mode。）

- "You try a thing, you give it to users, you talk to users, you learn, and then eventually you might end up at a good idea. Sometimes you don't."（你尝试一件事，把它给用户，与用户交流，你学习，然后最终你可能找到一个好主意。有时没有。）
