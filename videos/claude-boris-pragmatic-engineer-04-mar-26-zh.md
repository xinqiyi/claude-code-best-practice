## 核心要点

### Claude Code 当前状态

- Claude Code 编写了 Anthropic 平均约 80% 的代码
- Boris 每天写 10-20 个 PR，100% 由 Opus 4.5 和 Claude Code 编写
- 他不再手动编辑任何一行代码

### 印刷机类比

Boris 用 15 世纪的印刷机来类比当前 AI 编码的变革：

- 当时有会写字的抄写员（scribes）
- 有些国王是文盲，雇佣抄写员
- 抄写员这个职业消失了，但作家和作者（writers and authors）这个类别诞生了
- 文学市场大幅扩张——新类别应运而生

**对工程师的启示：** 工程师不会消失，但角色会转变。

### Boris 的编码起源

- 13 岁时在 eBay 上卖 Pokemon 卡，通过 HTML 让卡片卖得更好
- 用 TI83 绘图计算器编程序来解数学题
- 问题变难后，从 BASIC 降到 assembly 来让程序跑得更快

### 关键工程洞察

- 在 Anthropic 的第一个 PR 被拒绝了——因为他是手写的！
- 经理说："你手写的？你在干什么？用 Clyde（Claude Code 的前身）！"
- 模型改进太快，以至于"用旧模型有效的想法可能在新模型上不适用"
- "模型只是想要使用工具"——当给模型 bash 工具时，它开始写 AppleScript 来自动化事情

### Andrej Karpathy 的观察

- Karpathy 说他"作为程序员从未感到如此落后"
- Boris 也有同感——这是指数变化速度带来的挑战

## 关键引用

- "I wrote maybe 10-20 PRs every day. Opus 4.5 and Claude Code wrote 100% of every single one. I didn't edit a single line manually."（我每天大概写 10-20 个 PR。Opus 4.5 和 Claude Code 写了每一个的 100%。我没有手动编辑一行代码。）

- "One metaphor I have for this moment in time is the printing press in the 1400s."（我对这个时刻的一个比喻是 15 世纪的印刷机。）
