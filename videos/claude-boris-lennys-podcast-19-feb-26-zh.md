## 核心要点

### 关于 Claude Code 的起源

- Boris 从 2025 年 11 月起就不再手写代码了，100% 的代码由 Claude Code 编写
- 每天发布 10、20、30 个 PR
- Claude Code 起初只是一个"小 hack"——Boris 花了几个月时间在终端里做原型
- 第一个版本叫 QuadCLI，内部发布时只获得了 2 个赞
- 之所以在终端中构建，是因为"资源不足"——只有 Boris 一个人，终端是最简单的构建方式
- 最终终端形式被保留下来，因为模型进步太快——没有其他 form factor 能跟上

### 关于 Cursor 的短暂经历

- Boris 短暂加入过 Cursor（2 周），因为被 Cursor 团队的产品愿景所吸引
- 他很快意识到他真正想念的是 Anthropic 的使命（safety）
- "无论工作多么令人兴奋，即使是在构建一个非常酷的产品，也真的不能替代使命感"

### Claude Code 的增长

- Claude Code 占所有 GitHub 公开 commits 的 4%（SemiAnalysis 数据）
- 如果看私有仓库，这个比例更高
- 增长率仍在加速——"不是只在上升，而是在越来越快地上升"
- Anthropic 最近融资超过 $3500 亿

### 为什么 Coding 正在被"解决"

- "在一两年内，是否应该学习编程将不再重要。编码已经基本被解决了"
- Anthropic 的愿景路径：编码能力 → 工具使用 → 计算机使用
- "软件工程师这个头衔将开始消失。它将被 'builder' 取代"
- 所有人都在变成"产品经理"，而每个人也同时是编码者

### AI-Native 的开发方式

- Boris 在录制播客时有 5 个 agent 在后台同时运行
- Claude Code 开始不仅能写代码，还能提出想法——查看反馈、bug 报告和遥测数据
- "Claude Code 变得更像一个 co-worker"

### 关键建议

- "我们不为今天的模型构建。我们为 6 个月后的模型构建"
- 产品原则：undersource 你的项目——在开始时保持团队小
- 如果你想做好的 AI 工作，"你真的必须在一定程度上理解模型"
- 最好的信号来源是"用户的反馈"
