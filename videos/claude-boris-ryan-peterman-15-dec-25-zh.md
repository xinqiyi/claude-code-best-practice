## 文字记录

（以下为英文访谈转录的中文翻译概要）

本访谈涵盖了 Boris Cherny 从 Meta 到 Anthropic 的职业历程，以及他在此过程中学到的关键教训：

- **Latent Demand：** Boris 认为这是产品中最重要的原则——观察用户如何"滥用"你的产品，然后在已有意图的基础上构建功能。Facebook Marketplace 源自观察到 Facebook 群组中 40% 的帖子是买卖物品；Facebook Dating 源自观察到 60% 的个人资料浏览来自非好友关系的异性。

- **跨组织合作：** 在 Meta，将 Messenger 和 Facebook 群组结合的项目因两组织的价值观完全相反（Facebook 追求速度，Messenger 追求可靠性）而面临巨大困难。Boris 建议寻找双方都能受益的共同目标。

- **Side Quests 的重要性：** Boris 强调副项目对职业成长的重要性。他的 Undux 状态管理 framework 成为 Facebook 最受欢迎的状态管理方案。他还写了一本 TypeScript 书籍并组织了旧金山最大的 TypeScript 聚会。

- **Levels 和期望：** Boris 认为"被低定级"是一种幸运——更低的期望给了他探索和构建很酷的东西的空间。

- **技术领导力：** 作为 tech lead，Boris 建议使用"时间盒"方法进行 scoping（30 分钟到几小时），并向专家展示 straw man 方案以获取反馈，而不是仅仅征求输入。

- **Imposter Syndrome：** Boris 的建议是"不要想太多。没有人真正知道自己在做什么"。

- **"更好工程"作为影响力：** Boris 认为"better engineering 是作为工程师扩展人脉和获得影响力的最简单方式"。他提到每个 code review 中的重复问题都可以通过一个 lint rule 自动化。

- **从 Meta 到 Anthropic：** Boris 被 AI 的魔力所吸引。他选择 Anthropic 是因为其使命——确保 AI 发展走在正确的道路上——并且被那里的"硬核科幻迷"文化所吸引。

- **Claude Code 的创建：** Boris 的经理 Ben 推动他"为 6 个月后的模型构建，而不是为今天的模型构建"。产品在 Opus 4 发布后才真正起飞。现在 Anthropic 已增长三倍，但由于 Claude Code，每个工程师的生产力增长了近 70%。

- **生产力技巧：** Boris 现在的建议是"学习如何使用 Claude Code，学习如何运行一堆 Claude Code 来做事"。他描述了一个工程师在周末通过运行 20 个 Claude 实例 swarm 构建了 plugins 的完整功能。

- **职业建议：** Boris 的最后建议是"用常识"——大公司里有很多东西会让你远离常识，但最终相信自己和你的常识是最重要的。

## 关键引用

- "Don't build for the model of today. Build for the model 6 months from now."（不要为今天的模型构建，要为 6 个月后的模型构建）

- "Latent demand I think is the single most important principle in product."（latent demand 我认为是产品中最重要的原则）

- "The one technical book I would recommend to everyone that has had the greatest impact on me as an engineer is 'Functional Programming in Scala'."（我会向每个人推荐的一本对我作为工程师影响最大的技术书籍是《Functional Programming in Scala》）

- "You have to understand the trade-offs."（你必须理解权衡）

- "Delegate the thing that you like to do so then you can monitor progress."（委托你喜欢做的事，这样你才能监控进度）

- "Just use common sense."（用常识就好）
